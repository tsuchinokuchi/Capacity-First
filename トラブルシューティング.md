# 🔧 トラブルシューティング: QuickAddコマンドが表示されない

## 確認手順

### 1. QuickAddプラグインが有効か確認
1. 設定（⚙️）を開く
2. 「コミュニティプラグイン」を選択
3. 「QuickAdd」が有効になっているか確認
4. 無効の場合は「有効化」をクリック

### 2. Obsidianを再起動
- `Ctrl+R` (Windows) または `Cmd+R` (Mac) でリロード
- または、Obsidianを完全に終了して再起動

### 3. コマンドパレットで検索
- `Ctrl+P` (Windows) または `Cmd+P` (Mac) でコマンドパレットを開く
- 以下のキーワードで検索：
  - 「タスク追加」
  - 「タスク移動」
  - 「QuickAdd: タスク追加」
  - 「QuickAdd: タスク移動」

### 4. QuickAddの設定画面で確認
1. 設定（⚙️）→「QuickAdd」を選択
2. 「Choices」タブを開く
3. 「タスク追加」と「タスク移動」が表示されているか確認
4. 各Choiceの右側に「⚡」アイコンがあるか確認（コマンドが有効な証拠）

### 5. 手動でコマンドを有効化
もし上記で見つからない場合：
1. QuickAddの設定画面で「タスク追加」Choiceを選択
2. 設定項目の中に「Command」または「コマンド」のトグルがあるか確認
3. 有効になっているか確認

## 代替方法: 直接スクリプトを実行

コマンドパレットから実行できない場合、以下の方法で直接スクリプトを実行できます：

### 方法1: Templaterで実行
1. Templaterの設定でUser Scriptsフォルダを設定
2. スクリプトをUser Scriptsとして実行

### 方法2: コマンドパレットからスクリプトを直接実行
1. `Ctrl+P` でコマンドパレットを開く
2. 「Script:」と検索
3. 「タスク追加」または「タスク移動」を選択

## 設定ファイルの確認

設定ファイル（`.obsidian/plugins/quickadd/data.json`）で以下を確認：
- `"command": true` になっているか
- スクリプトのパスが正しいか（`.obsidian/scripts/タスク追加.js`）

## それでも解決しない場合

1. QuickAddプラグインを無効化→有効化
2. Obsidianを完全に再起動
3. エラーログを確認（`Ctrl+Shift+I` でコンソールを開く）
