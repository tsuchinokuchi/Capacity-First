# ✅ 動作確認チェックリスト

## 1. プロジェクト管理機能（リファクタリング後）

### テストケース1: プロジェクト作成
1. [ ] QuickAddから「プロジェクト作成」を実行
2. [ ] プロジェクト名、概要、期間を入力
3. [ ] 作成されたプロジェクトファイルを開く
4. [ ] 「進捗」セクションと「サブタスク」セクションが表示されていることを確認
5. [ ] ボタン（今月バージョン作成、サブタスク追加、日付を設定）が表示されていることを確認

### テストケース2: サブタスク追加ボタン
1. [ ] プロジェクトファイル内の「➕ サブタスク追加」ボタンをクリック
2. [ ] タスク名、ジャンル、時間、日付を入力
3. [ ] カテゴリを選択（または新規作成）
4. [ ] プロジェクトファイル内の指定カテゴリにタスクが追加されることを確認
5. [ ] 指定した日付のスケジュールファイルにもタスクが追加されることを確認

### テストケース3: 日付を設定ボタン
1. [ ] プロジェクトファイル内のタスクのチェックボックスをいくつか選択（未完了タスク）
2. [ ] 「📅 日付を設定」ボタンをクリック
3. [ ] ポップアップで対象タスクを選択
4. [ ] 新しい日付を入力
5. [ ] プロジェクトファイル内のタスクの日付が更新されることを確認
6. [ ] 新しい日付のスケジュールファイルにタスクが追加されることを確認
7. [ ] （もしあれば）古い日付のスケジュールファイルからタスクが削除されることを確認

### テストケース4: 進捗表示
1. [ ] プロジェクトファイル内のタスクを完了（`[x]`）にする
2. [ ] Dataviewの更新を待つ（またはコマンドでリロード）
3. [ ] 進捗バーとパーセンテージが更新されることを確認

## 2. タスク管理ダッシュボード（リファクタリング後）

### テストケース1: ダッシュボード表示
1. [ ] `タスク管理ダッシュボード.md` を開く
2. [ ] 以下のセクションが正しく表示されることを確認
    - 📊 Capacity-first Task Dashboard（タイトル）
    - ⏱️ 日付と自動更新
    - 🗂 タスクプール & 期限切れ（折りたたみ）
    - ✅ 今日のタスク
    - 🚀 プロジェクト進捗（折りたたみ）
    - 📅 明日のタスク（折りたたみ）
    - 📆 今週のサマリー（折りたたみ）
    - 📎 ショートカット

### テストケース2: 今日のタスク操作
1. [ ] 「✅ 今日のタスク」セクションでタスクを追加ボタンをクリック -> QuickAddが起動することを確認
2. [ ] タスクのチェックボックスをオンにする -> 完了状態になることを確認
3. [ ] タスクを選択して「完了」「削除」「日付変更」ボタンが動作することを確認

### テストケース3: タスクプール操作
1. [ ] 「🗂 タスクプール & 期限切れ」を展開
2. [ ] プールにあるタスク（または期限切れタスク）が表示されていることを確認
3. [ ] タスクを選択して「日付変更」を実行 -> 指定した日のスケジュールに移動することを確認

### テストケース4: プロジェクト進捗一覧
1. [ ] 「🚀 プロジェクト進捗」を展開
2. [ ] 進行中のプロジェクト一覧と進捗バーが表示されていることを確認
3. [ ] プロジェクト名をクリックしてファイルが開くことを確認
