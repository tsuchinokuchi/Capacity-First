# ✅ 動作確認チェックリスト

## 1. タスク追加機能の確認

### テストケース1: 基本的なタスク追加
1. [ ] `Ctrl+P` (または `Cmd+P`) でコマンドパレットを開く
2. [ ] 「QuickAdd: タスク追加」を検索して実行
3. [ ] タスク名を入力（例：「テストタスク」）
4. [ ] 所要時間を選択（例：「60分」）
5. [ ] ジャンルを選択（例：「デスクワーク」）
6. [ ] 日付を入力（デフォルト：今日、そのままEnterでOK）
7. [ ] スケジュールファイルにタスクが追加されることを確認
8. [ ] 成功メッセージが表示されることを確認

**期待される結果:**
- タスクが `スケジュール/YYYY-MM-DD.md` に追加される
- フォーマット: `- [ ] テストタスク #デスクワーク ⏱️ 60 📅 2025-11-01`
- 容量使用状況が表示される

### テストケース2: 容量チェック（正常）
1. [ ] 今日のタスクが少ない状態で、新しいタスクを追加
2. [ ] 容量が300分を超えない場合、警告なしで追加されることを確認

### テストケース3: 容量チェック（超過）
1. [ ] 今日のタスクを複数追加して、合計が300分を超えるようにする
2. [ ] 容量超過警告が表示されることを確認
3. [ ] 「はい」を選択した場合、タスクが追加されることを確認
4. [ ] 「いいえ」を選択した場合、タスクが追加されないことを確認

### テストケース4: 日付指定
1. [ ] 今日以外の日付（例：明日）を指定してタスクを追加
2. [ ] 指定した日付のスケジュールファイルに追加されることを確認

## 問題が発生した場合

1. **エラーメッセージを確認**
   - Obsidianのコンソール（`Ctrl+Shift+I`）を開く
   - エラーメッセージをコピー

2. **スクリプトのパスを確認**
   - `.obsidian/scripts/` フォルダにスクリプトがあるか確認
   - ファイル名が正しいか確認

3. **QuickAddの設定を確認**
   - User Scriptが正しく追加されているか確認
   - ChoiceのTypeが「Macro」になっているか確認
